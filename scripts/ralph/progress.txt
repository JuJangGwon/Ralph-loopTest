## Codebase Patterns
- Server entry: `server/index.js` — CommonJS, dotenv loaded from project-root `.env` via `path.resolve(__dirname, '..', '.env')`
- Run server: `cd server && npm start` (typecheck via `npm run typecheck` = `node --check` over each .js file)
- CORS origin allowlist: `http://localhost:5173` (Vite client default)
- `app` exported from `server/index.js`; `app.listen()` only runs when invoked as main module — keeps app importable from tests
- Two `package.json`s expected: `server/` (CommonJS) and future `client/` (Vite). No root package.json yet.
- `.env.example` lives at project root, NOT inside server/ (single source of truth for both server and future client config)
- Benefits cache: `server/benefits.js` exposes `loadBenefits()` (boot-time fetch + pagination) and `getBenefits()` (sync accessor). Module-scoped `BENEFITS` array; fallback `[]` on error. `_setBenefitsForTest(list)` available for unit tests (US-003 will use it).
- Normalized benefit shape: `{ id, name, region, jobCondition, incomeCondition: { earnMinAmt, earnMaxAmt, earnCndSeCd, raw }, summary, applyUrl }` — region uses `zipCd` from API (numeric region code). Future scoring/UI must compare regions in the same code space, NOT raw Korean names.
- 온통청년 API: JSON endpoint `https://www.youthcenter.go.kr/go/ythip/getPlcy`, params `apiKeyNm/pageNum/pageSize/rtnType=json`, response `result.youthPolicyList` + `result.pagging.totCount`. Override via `YOUTH_API_URL` env if needed.
- When `YOUTH_API_KEY` empty, `loadBenefits()` short-circuits to `[]` with a warning — server still boots cleanly. Use this for offline dev.
