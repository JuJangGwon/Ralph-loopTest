## Codebase Patterns
- Server entry: `server/index.js` — CommonJS, dotenv loaded from project-root `.env` via `path.resolve(__dirname, '..', '.env')`
- Run server: `cd server && npm start` (typecheck via `npm run typecheck` = `node --check` over each .js file)
- CORS origin allowlist: `http://localhost:5173` (Vite client default)
- `app` exported from `server/index.js`; `app.listen()` only runs when invoked as main module — keeps app importable from tests
- Two `package.json`s expected: `server/` (CommonJS) and future `client/` (Vite). No root package.json yet.
- `.env.example` lives at project root, NOT inside server/ (single source of truth for both server and future client config)
- Benefits cache: `server/benefits.js` exposes `loadBenefits()` (boot-time fetch + pagination) and `getBenefits()` (sync accessor). Module-scoped `BENEFITS` array; fallback `[]` on error. `_setBenefitsForTest(list)` available for unit tests (US-003 will use it).
- Normalized benefit shape: `{ id, name, region, jobCondition, incomeCondition: { earnMinAmt, earnMaxAmt, earnCndSeCd, raw }, summary, applyUrl }` — region uses `zipCd` from API (numeric region code). Future scoring/UI must compare regions in the same code space, NOT raw Korean names.
- 온통청년 API: JSON endpoint `https://www.youthcenter.go.kr/go/ythip/getPlcy`, params `apiKeyNm/pageNum/pageSize/rtnType=json`, response `result.youthPolicyList` + `result.pagging.totCount`. Override via `YOUTH_API_URL` env if needed.
- When `YOUTH_API_KEY` empty, `loadBenefits()` short-circuits to `[]` with a warning — server still boots cleanly. Use this for offline dev.
- Test runner: `node --test <file>` (built-in, no deps). Pass an explicit file path — `node --test __tests__` resolves the directory as a CJS module on Windows and crashes (MODULE_NOT_FOUND). Run via `npm test` from `server/`.
- Scoring: `server/score.js` exposes pure `scoreBenefit(benefit, input)` and `recommend(benefits, input)`. Region-match treats `''/null/'전국'/'00000'/'003001'` as nationwide (+1); exact string equality is +3. Job match splits benefit.jobCondition on `,/|`, then substring fallback. Income +2 when min/max both null OR user income within [min,max]. Recommend filters score<1, sorts score desc, name asc tiebreak.
- POST /api/recommend handler in `index.js` 400s with `{ error: 'missing required fields: ...' }` when any of name/region/job/income absent (income==='' counts as missing). Otherwise returns `{ recommendations }`.
- Client root: `client/` — Vite 5 + React 18 + react-router-dom 6, scaffolded manually (no `npm create vite` to avoid interactive prompts). `package.json type: "module"`, JSX in `.jsx` files, dev port 5173 strictPort.
- Client routes wired in `client/src/App.jsx` (Routes/Route) under `BrowserRouter` from `main.jsx`. Pages live in `client/src/pages/*.jsx`. Use `data-route` attribute on placeholder pages for sanity checks.
- Client typecheck: `npm run typecheck` runs `node --check vite.config.js`. Extend script per file when adding TS/JS source files that need static syntax check (parallels server pattern).
- Client env: `VITE_API_BASE_URL` from `client/.env` (committed) + `client/.env.example`. Vite only exposes vars prefixed `VITE_`.
- Root `.gitignore` already covers `node_modules/` and `dist/` for both `server/` and `client/` — no client-specific ignore needed.
- Client state: `client/src/store/funnelStore.js` exports `useFunnelStore` (zustand). Fields `name/region/job/income` default `''`. Mutators: `setField(field, value)` and `reset()`. Funnel pages should call `useFunnelStore((s) => s.field)` for narrow subscriptions and `setField` for writes — never mutate state directly.
- Client typecheck pattern: append `&& node --check <path>` to the `typecheck` script for every new `.js` source file (parallels server). JSX files skipped — `node --check` cannot parse JSX.
